## Story

**As a** membro da equipe interna,
**I want** criar e alternar entre projetos no header do dashboard,
**so that** eu possa organizar tracking por produto/curso separadamente.

---

## Acceptance Criteria

1. Layout shell implementado: header com logo, project switcher (dropdown), user menu (avatar + logout)
2. Sidebar com nav items: Dashboard, Links, Conversoes (disabled/placeholder por enquanto)
3. Project switcher lista todos os projetos do usuario e permite alternar — projeto ativo salvo em cookie
4. Modal de "Novo Projeto" com campos: nome, slug (auto-gerado), descricao, URL base
5. CRUD completo de projetos via API routes (`/api/projects`)
6. Pagina `/` exibe estado vazio ("Nenhum dado ainda") quando projeto nao tem dados
7. Projeto ativo propagado via Zustand store para todos os componentes filhos
8. Responsivo: sidebar colapsa em bottom nav no mobile

### Database Schema [Source: Story 1.2]

```sql
CREATE TABLE public.projects (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name        TEXT NOT NULL,
  slug        TEXT NOT NULL UNIQUE,
  description TEXT,
  base_url    TEXT NOT NULL,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

RLS: All authenticated users can CRUD (MVP equipe interna).

### API Endpoints [Source: architecture.md#5.3]

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/projects` | Lista todos os projetos |
| POST | `/api/projects` | Cria novo projeto |
| GET | `/api/projects/[id]` | Detalhe |
| PATCH | `/api/projects/[id]` | Atualiza |
| DELETE | `/api/projects/[id]` | Remove |

POST body: `{ name: string, slug?: string, description?: string, base_url: string }`
