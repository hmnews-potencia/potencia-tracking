# Story 3.2: Tracking Ingestion API

---

## Status

**Done**

---

## Story Statement

As a member of the internal team, I want a tracking API endpoint that receives events from the tracking script, so that pageviews, clicks, and conversions are stored in the database for analytics.

---

## Acceptance Criteria

1. `POST /api/track` recebe eventos com payload: project_slug, tracking_id, event_type, timestamp, utm_*, page_url, referrer, user_agent, conversion_data?
2. Validacao: project_slug deve existir, event_type = pageview|click|conversion, UTM params sanitizados
3. IP hasheado (SHA-256 + salt) antes de salvar — nunca armazena IP raw (LGPD)
4. CORS configurado: aceita requests de qualquer origem
5. Resposta rapida: retorna 202 Accepted imediatamente
6. Conversoes com tracking_id existente sao associadas ao evento original
7. Conversoes sem tracking_id previo criam registro "orfao" com flag is_orphan: true
8. Endpoint `GET /api/track/health` retorna status da API
